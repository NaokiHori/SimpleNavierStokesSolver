Initialise a structure param_t *param and its members.

param_init¶

Definition¶

int param_init(
    param_t **param
);

Name

intent

description

param

out

allocated and initialised

Description¶

Note

Mainly, this function takes care of the following things.

  1. Allocate a param_t structure

  2. Load and initialise parameters (e.g., domain size, number of grid points, maximum simulation time)

  3. Initialise coordinate system (e.g., param->xf) and grid sizes (e.g., param->dxf, dy)

  4. Initialise coefficients which are used by the three-step Runge-Kutta scheme

First of all, a param structure is allocated:

239  param_t *param = common_calloc(1, sizeof(param_t));

param structure contains coordinates, which cannot be allocated at this point since the number of grid points (e.g., param->itot and param->jtot) are unknown.

The next step is to load parameters (environmental variables, such as domain size) to configure:

241  load_config(param);

which is followed by allocating and initialising coordinates:

243  set_coordinate(param);

The simulation can be restarted by loading previous flow field when desired, in particular when an environmental variable dirname_restart (in which the flow field is contained) is given. When we do so, step and time are loaded, otherwise 0 is assigned:

245  if(param->load_flow_field){
246    fileio_r_0d_serial(param->dirname_restart, "step", sizeof(int),    &(param->step));
247    fileio_r_0d_serial(param->dirname_restart, "time", sizeof(double), &(param->time));
248  }else{
249    param->step = 0;
250    param->time = 0.;
251  }

Some procedures are outsourced to the other functions, whose detailed descriptions can be found below.

load_config¶

Definition¶

static int load_config(
    param_t *param
);

Name

intent

description

param

out

members are initialised

Description¶

Members of the param structure are initialised, where values from environmental variables (normally a configuration file exec.sh is used to define them for convenience) are assigned. When a parameter is not given (the corresponding environmental variable is not defined), a default value is used.

In particular, this function tries to load a variable "dirname_restart", which is used to designate the name of the directory which contains data of the intermediate flow field. If given, a flag param->load_flow_field is set to true and the simulator tries to load the intermediate velocity, pressure, and temperature fields from the files later. Otherwise all these fields are initialised later.

91  param->dirname_restart = load_env_as_string("dirname_restart");
92  if(param->dirname_restart == NULL){
93    param->load_flow_field = false;
94    PRINTF_MAIN("  flow fields are initialised\n");
95  }else{
96    param->load_flow_field = true;
97    PRINTF_MAIN("  flow fields are loaded: %s\n", param->dirname_restart);
98  }

Other parameters are also loaded.

  1. Flags to decide the temperature coupling:

    100  param->with_temperature     = load_bool("with_temperature",     true);
101  param->with_thermal_forcing = load_bool("with_thermal_forcing", true);

    Note that the second argument is a default value, which is used when the parameter is not given.

  2. Temporal information (e.g., maximum simulation time, rate to output logs, to save flow fields, and to collect statistics):

    103  param->timemax    = load_double("timemax",    2.0e+2);
104  param->wtimemax   = load_double("wtimemax",   6.0e+2);
105  param->log.rate   = load_double("log_rate",   2.0e-1);
106  param->log.after  = load_double("log_after",  0.0e+0);
107  param->save.rate  = load_double("save_rate",  2.0e+1);
108  param->save.after = load_double("save_after", 0.0e+0);
109  param->stat.rate  = load_double("stat_rate",  1.0e-1);
110  param->stat.after = load_double("stat_after", 1.0e+2);

  3. Spatial information:

    112  param->itot    = load_int("itot", 128);
113  param->jtot    = load_int("jtot", 192);
114  param->lx      = LX; // fixed to unity
115  param->ly      = load_double("ly", 2.0e+0);
116  param->stretch = load_double("stretch", 3.0e+0);

  4. Non-dimensional parameters governing the physics (see the governing equations)

    118  param->Ra      = load_double("Ra", 1.0e+8);
119  param->Pr      = load_double("Pr", 1.0e+1);

See param for the detailed explanation of all members.

set_coordinate¶

Definition¶

static int set_coordinate(
    param_t *param
);

Name

intent

description

param

out

coordinates members are initialised

Description¶

The structure param has members defining the locations of grid points (param->xf, param->xc) and the distances between two grid points (param->dxf, param->dxc, and param->dy), which are initialised in this function.

First of all, the vectors to store the coordinates are allocated:

130  param->xf  = common_calloc(1, XF_MEMSIZE);
131  param->xc  = common_calloc(1, XC_MEMSIZE);
132  param->dxf = common_calloc(1, DXF_MEMSIZE);
133  param->dxc = common_calloc(1, DXC_MEMSIZE);
134  param->yf  = common_calloc(1, YF_MEMSIZE);
135  param->yc  = common_calloc(1, YC_MEMSIZE);

Coordinate generation process in the wall-normal direction (non-uniform grid spacings are allowed) is as follows.

  1. XF: cell-face positions

    Coordinate creation starts by defining the cell-face locations (param->xf: \(x\) locations where \(\ux\) are positioned):

    Click here for details

    In order to resolve the boundary layers close to the walls, more grid points can be clustered in the vicinity of the walls. By default, it is based on a cosine function whose edges are clipped (clipped Chebyshev).

    The following image shows how the coordinates are changed by manipulating str, where itot = 8.

    We notice that, for str = 0 (red line), XF does not change rapidly (as a function of i) in the vicinity of the boundaries, indicating that the grid is gathered there. For str = 4 (green line), on the other hand, XF changes almost linearly (as a function of i), telling that the cell boundaries XF are almost uniformly distributed.

    As can be seen in the figure, XF changes from 1 to -1, which should be transformed to a coordinate starting from 0 and ends at lx.

    The full implementation can be found here:

    143  {
144    int stretch = param->stretch;
145    if(stretch < 0){
146      stretch = 0;
147    }
148    if(stretch >= itot/2){
149      // uniform grid
150      const double dx = lx/itot;
151      for(int i=1; i<=itot+1; i++){
152        XF(i) = 1.*(i-1)*dx;
153      }
154    }else{
155      // clipped Chebyshev
156      for(int i=1; i<=itot+1; i++){
157        XF(i) = cos(M_PI*(1.*(i-1)+stretch)/(itot+2.*stretch));
158      }
159      // original XF: "1"->"-1", so we need to normalise
160      double min = XF(     1);
161      double max = XF(itot+1);
162      for(int i=1; i<=itot+1; i++){
163        XF(i) = lx*(XF(i)-min)/(max-min);
164      }
165    }
166    // force boundary values just in case
167    XF(     1) = 0.;
168    XF(itot+1) = lx;
169  }

    Of course users can adopt other functions if needed to change the behaviour.

    Note

    str denotes the number of clipped points at both edges. Thus it is non-sense to give larger value than itot/2. In this case, uniform grid is created instead.

  2. DXF: face-to-face distance

    Since the cell face positions are known now, we initialise the distances between cell face to face (param->dxf, \(x\) locations where \(u_x\) are positioned):

    The implementation can be found here:

    171  for(int i=1; i<=itot; i++){
172    DXF(i) = XF(i+1)-XF(i  );
173  }

  3. XC: cell-center positions

    Cell centers (param->xc, \(x\) positions where \(p\) and \(u_y\) are located) are defined in the middle of the two neighbouring cell faces:

    The implementation can be found here:

    175  // at boundaries, face positions are assigned
176  XC(0) = XF(1);
177  for(int i=1; i<=itot; i++){
178    XC(i) = 0.5*(XF(i)+XF(i+1));
179  }
180  XC(itot+1) = XF(itot+1);

  4. DXC: center-to-center distance

    Finally the distance between cell centers are computed:

    The implementation can be found here:

    182  // at boundaries, face-center distances are assigned
183  for(int i=1; i<=itot+1; i++){
184    DXC(i) = XC(i)-XC(i-1);
185  }

    Note that a special treatment is needed at the edges; cell centers (and variables defined there, e.g., pressure and temperature) are shifted to cell faces at the boundaries.

Regarding the grid size in \(y\) direction (only uniform grid spacings are possible), it is simply given as:

187  param->dy = ly/jtot;
188  for(int j=1; j<=jtot+1; j++){
189    YF(j) = 1.*(j-1)*param->dy;
190  }
191  for(int j=1; j<=jtot; j++){
192    YC(j) = 0.5*(2*j-1)*param->dy;
193  }

since \(y\) direction only allows uniform grid spacings for now.

Note

YC and YF are not used in the main integration. They exist just for completeness.
