and values are initialised when the simulation is started from the beginning, or values are loaded from a directory (char param->dirname_restart, see param_t) to restart the run:

90  init_or_load(param, parallel, fluid);

Description¶

Velocity and pressure fields are initialised.

When we want to start the simulation from the beginning (i.e., param->load_flow_field is false), initial conditions are assigned here:

46    for(int j=1; j<=jsize; j++){
47      for(int i=2; i<=itot; i++){
48        UX(i, j) = 0.;
49      }
50    }

60    for(int j=1; j<=jsize; j++){
61      for(int i=1; i<=itot; i++){
62        UY(i, j) = 0.;
63      }
64    }

74    for(int j=1; j<=jsize; j++){
75      for(int i=1; i<=itot; i++){
76        P(i, j) = 0.;
77      }
78    }

Otherwise (i.e., param->load_flow_field is true), values are loaded from a restart file:

43    fileio_r_ux_like_parallel(param->dirname_restart, "ux", param, parallel, fluid->ux);

57    fileio_r_uy_like_parallel(param->dirname_restart, "uy", param, parallel, fluid->uy);

71    fileio_r_p_like_parallel(param->dirname_restart, "p", param, parallel, fluid->p);

where flow fields are loaded from npy files (see src/fileio.c for details).

Note that flow fields excluding halo cells are set here, and these edge values are set subsequently:

53  fluid_update_boundaries_ux(param, parallel, ux);

67  fluid_update_boundaries_uy(param, parallel, uy);

81  fluid_update_boundaries_p(param, parallel, p);
