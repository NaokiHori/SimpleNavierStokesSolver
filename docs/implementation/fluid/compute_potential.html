fluid/compute_potential.c¶

Compute a scalar potential \(\psi\) which projects a non-solenoidal velocity field to a solenoidal one by solving a Poisson equation

\[\frac{\delta^2 \psi}{\delta x_i \delta x_i} = \frac{1}{\gamma \Delta t} \frac{\delta u_i^*}{\delta x_i}.\]

The methodology is discussed here.

Description¶

This function solves the Poisson equation described above.

First of all, we compute the right-hand-side values and store them to a variable qrx (x -aligned r eal q uantity), which is allocated in the function as well as other buffers:

34    // total size in x direction and its MPI-decomposed size
35    const int xsize_total  =                   itot;

37    // total size in y direction (varying in physical and wave spaces) and their MPI-decomposed sizes
38    const int ysize_total_physical  =                   jtot;
39    const int ysize_decomp_physical = parallel_get_size(jtot, mpisize, mpirank);
40    const int ysize_total_wave      =                   jtot/2+1;
41    const int ysize_decomp_wave     = parallel_get_size(jtot/2+1, mpisize, mpirank);
42    // datasizes
43    size_t size_r = sizeof(double);
44    size_t size_c = sizeof(fftw_complex);
45    // in real (physical) space, x-aligned (decomposed in y)
46    qrx = common_calloc(xsize_total  * ysize_decomp_physical, size_r);
47    // in real (physical) space, y-aligned (decomposed in x)
48    qry = common_calloc(xsize_decomp * ysize_total_physical,  size_r);
49    // in complex (wave) space, x-aligned (decomposed in y)
50    qcx = common_calloc(xsize_total  * ysize_decomp_wave,     size_c);
51    // in complex (wave) space, y-aligned (decomposed in x)
52    qcy = common_calloc(xsize_decomp * ysize_total_wave,      size_c);

Note that the size of this variable is itot \(\times\) jsize = parallel_get_size(jtot, mpisize, mpirank), i.e., no halo cells are included.

qrx is defined at the same location as pressure (i.e., cell center) and thus the discrete equation leads

\[\left( qrx \right)_{i, j} = \frac{1}{\gamma \Delta t} \left( \frac{\left. u_x \right|_{i+\frac{1}{2},j} - \left. u_x \right|_{i-\frac{1}{2},j}}{\Delta x_i} + \frac{\left. u_y \right|_{i,j+\frac{1}{2}} - \left. u_y \right|_{i,j-\frac{1}{2}}}{\Delta y } \right).\]

In the code, it is implemented as

70  const double gamma = param->rkcoefs[rkstep].gamma;
71  const double dt = param->dt;
72  const double *ux = fluid->ux;
73  const double *uy = fluid->uy;
74  for(int j=1; j<=jsize; j++){
75    for(int i=1; i<=itot; i++){

77      double ux_xp = UX(i+1, j  );
78      double uy_ym = UY(i  , j  );
79      double uy_yp = UY(i  , j+1);
80      QRX(i, j) =
81        1./(gamma*dt)*(
82         +(ux_xp-ux_xm)/DXF(i)
83         +(uy_yp-uy_ym)/dy
84        );
85    }
86  }

Note that halo values are used here (UY(i, jsize+1)), which should be communicated beforehand (at the end of fluid/update_velocity.c <fluid_update_velocity>).

Since we adopt the second-order accurate central finite difference scheme in space, the Poisson equation is discretised as

\[\frac{\frac{\psi_{i+1,j} - \psi_{i,j}}{\Delta x_{i+\frac{1}{2}}} - \frac{\psi_{i,j} - \psi_{i-1,j}}{\Delta x_{i-\frac{1}{2}}}}{\Delta x_i} + \frac{\psi_{i,j+1} - 2 \psi_{i,j} + \psi_{i,j-1}}{\Delta y^2} = \left( qrx \right)_{i,j}.\]

Recall that we consider a domain which is periodic in \(y\) direction, while the grid size can vary in \(x\) direction.

As we can see in the left-hand-side, this equation contains 5 different \(P\) values, which reduces to a sparse matrix with size \(\left( \text{itot} \times \text{jtot} \right)^2\) and a bit tedious to solve as it is.

In order to avoid this, here we consider the Discrete Fourier transform (a notation \(\mathcal{F}^D\) is used hereafter) of a quantity \(q_j\) in \(y\) direction, which is defined and computed as

\[Q_J \equiv \mathcal{F}^D \left[ q_j \right] = \sum_{j=1}^{\text{jtot}} q_j \exp{\left\{ -2 \pi \left( j - 1 \right) J \sqrt{-1} / \left( \text{jtot} \right) \right\}},\]

following the implementation of FFTW3. Here \(J\) is used as a index in the (\(y\) direction) wavespace, \(J \in \left[ 0 : \text{jtot}/2 \right]\). Hereafter capital letters in mathematical equations (not code) imply that the quantity is transformed to the corresponding wavespace.

Since

\[\mathcal{F}^D \left[ \psi_{i,j\pm1} \right] = \mathcal{F}^D \left[ \psi_{i,j} \right] \exp{\left\{ \pm 2 \pi J \sqrt{-1} / \left( \text{jtot} \right) \right\}}\]

holds as long as the signal is periodic in \(y\) direction, the original discrete Poisson equation can be written as

\[\frac{\frac{\Psi_{i+1,J} - \Psi_{i,J}}{\Delta x_{i+\frac{1}{2}}} - \frac{\Psi_{i,J} - \Psi_{i-1,J}}{\Delta x_{i-\frac{1}{2}}}}{\Delta x_i} + \frac{\exp{\left\{ \frac{2 \pi J \sqrt{-1}}{\left( \text{jtot} \right)} \right\}}\Psi_{i,J} - 2 \Psi_{i,J} + \exp{\left\{ \frac{-2 \pi J \sqrt{-1}}{\left( \text{jtot} \right)} \right\}}\Psi_{i,J}}{\Delta y^2} = \left( QRX \right)_{i,J},\]

or by using the relation

\[\exp{ \left( 2 \theta \sqrt{-1} \right) } + \exp{ \left( -2 \theta \sqrt{-1} \right) } - 2 = 2 \cos \left( 2 \theta \right) - 2 = -4 \sin^2 \theta,\]

we have

\[\begin{split}\begin{aligned} \frac{1}{\Delta x_{i+\frac{1}{2}} \Delta x_i} \Psi_{i+1,J} & + \left( - \frac{1}{\Delta x_{i+\frac{1}{2}} \Delta x_i} - \frac{1}{\Delta x_{i-\frac{1}{2}} \Delta x_i} - \frac{4}{\Delta y^2} \sin^2 \left( \pi J \right) \right) \Psi_{i,J} + \frac{1}{\Delta x_{i-\frac{1}{2}} \Delta x_i} \Psi_{i-1,J} \\ & = \left( QRX \right)_{i,J}. \end{aligned}\end{split}\]

Note that the right-hand-side is also transformed (from \(\left( qrx \right)_{i,j}\) to \(\left( QRX \right)_{i,J}\))

Here only three neighbouring values \(\Psi_{i-1,J}\), \(\Psi_{i,J}\), and \(\Psi_{i+1,J}\) exist, i.e., \(J\)-variations (i.e., \(J+1\) and \(J-1\)) are gone and what we need to consider is an independent system for each \(J\). This is the primary advantage of using Fourier transform.

In short, by adopint the discrete Fourier transform in \(y\) direction, the original Poisson equation can be simplified.

As we see in the definition of the discrete Fourier transform, however, for each i, all \(\left( qrx \right)_{i, j}\) values from j=1 to j=jtot (not jsize) are needed to perform this computation (\(\because \sum_{j=1}^{\text{jtot}}\)), which cannot be done since our domain is split in \(y\) direction.

In order to resolve this issue, we first transpose the original variable qrx (decomposed in \(y\) direction) to a new variable qry (decomposed in \(x\) direction, y -aligned r eal q uantity):

88  parallel_transpose(itot, jtot, sizeof(double), MPI_DOUBLE, qrx, qry);

whose schematic can be found here:

where \(x\)-aligned variable (split in \(y\) direction) qrx in the left figure is transposed to a \(y\)-aligned variable (split in \(x\) direction) qry in the right figure.

Now each processor contains information aligned in \(y\) direction, which is followed by the discrete Fourier transform (forward transform) in \(y\) direction by calling a function offered by FFTW3 library:

90  fftw_execute(fwrd);

where fwrd is a pointer to the pre-defined plan, which is initialised a priori:

56    dims[0].is = 1;
57    dims[0].os = 1;
58    // forward transform in y direction, real to complex, physical to wave spaces

60    hdims[0].is = ysize_total_physical;

63    // backward transform in y direction, complex to real, wave to physical spaces
64    hdims[0].n  = xsize_decomp;
65    hdims[0].is = ysize_total_wave;
66    hdims[0].os = ysize_total_physical;

Here guru interface is adopted for simplicity, which executes multiple discrete Fourier transforms in \(y\) direction.

By this operation, y -aligned r eal q uantity qry is transformed to the y -aligned c omplex q uantity qcy, whose size is parallel_get_size(itot, mpisize, mpirank) \(\times\) jtot/2+1 (size in \(y\) direction is halved) because of the Hermitian symmetry, whose concept is drawn here:

where the size of the whole matrix shrinks in \(y\) direction by transforming qry to qcy.

Now, we have a equation which is independent of \(y\) (i.e., only \(J\) is involved, not \(J+1\) nor \(J-1\)); for convenience the equation is re-displayed here:

\[l_i \Psi_{i-1,J} + c_i \Psi_{i,J} + u_i \Psi_{i+1,J} = \left( QCX \right)_{i,J}.\]

where we defined

\[\begin{split}\begin{aligned} l_i & \equiv \frac{1}{\Delta x_{i-\frac{1}{2}} \Delta x_i}, \\ u_i & \equiv \frac{1}{\Delta x_{i+\frac{1}{2}} \Delta x_i}, \\ c_i & \equiv - \frac{1}{\Delta x_{i+\frac{1}{2}} \Delta x_i} - \frac{1}{\Delta x_{i-\frac{1}{2}} \Delta x_i} - \frac{4}{\Delta y^2} \sin^2 \left( \pi J \right), \\ & = - l_i - u_i - \frac{4}{\Delta y^2} \sin^2 \left( \pi J \right) \end{aligned}\end{split}\]

for simplicity. See the schematic below for the used grid arrangement:

This can be written as a linear system for each \(J\):

\[\begin{split}\begin{bmatrix} c_1 & u_1 & \cdots & 0 & 0 & 0 & \cdots & 0 & 0 \\ l_2 & c_2 & \cdots & 0 & 0 & 0 & \cdots & 0 & 0 \\ \vdots & \vdots & \ddots & \vdots & \vdots & \vdots & & \vdots & \vdots \\ 0 & 0 & \cdots & c_{i-1} & u_{i-1} & 0 & \cdots & 0 & 0 \\ 0 & 0 & \cdots & l_{i } & c_{i } & u_{i } & \cdots & 0 & 0 \\ 0 & 0 & \cdots & 0 & l_{i+1} & c_{i+1} & \cdots & 0 & 0 \\ \vdots & \vdots & & \vdots & \vdots & \vdots & \ddots & \vdots & \vdots \\ 0 & 0 & \cdots & 0 & 0 & 0 & \cdots & c_{\text{itot}-1} & u_{\text{itot}-1} \\ 0 & 0 & \cdots & 0 & 0 & 0 & \cdots & l_{\text{itot} } & c_{\text{itot} } \end{bmatrix} \begin{bmatrix} \Psi_{1, J} \\ \Psi_{2, J} \\ \vdots \\ \Psi_{i-1, J} \\ \Psi_{i , J} \\ \Psi_{i+1, J} \\ \vdots \\ \Psi_{\text{itot}-1, J} \\ \Psi_{\text{itot} , J} \end{bmatrix} = \begin{bmatrix} \left( QCX \right)_{1, J} \\ \left( QCX \right)_{2, J} \\ \vdots \\ \left( QCX \right)_{i-1, J} \\ \left( QCX \right)_{i , J} \\ \left( QCX \right)_{i+1, J} \\ \vdots \\ \left( QCX \right)_{\text{itot}-1, J} \\ \left( QCX \right)_{\text{itot} , J} \end{bmatrix},\end{split}\]

where the first matrix in the left-hand-side is the tri-diagonal matrix, which can be solved by the tri-diagonal matrix algorithm. See src/linalg.c for details.

In this code, two diagonal values (u pper- and l ower-diagonals) are stored as:

117      for(int i=1; i<=itot; i++){
118        TDM_L(i) = 1./DXC(i  )/DXF(i  );
119        TDM_U(i) = 1./DXC(i+1)/DXF(i  );
120      }

Another (c enter) diagonal values are assigned as

122      for(int i=1; i<=itot; i++){
123        TDM_C(i) = -TDM_L(i)-TDM_U(i)+eigenvalue;
124      }

where eigenvalue is

112      double eigenvalue = -4./pow(dy, 2.)*pow(
113          sin(M_PI*(j+joffset-1)/(1.*jtot)),
114          2.

Note that an offset joffset is added since J should be global. Also \(1\) is subtracted since J starts from 0.

Additional care should be taken for the boundary treatment.

At \(x = 0\), we have

\[l_1 \Psi_{0,J} + c_1 \Psi_{1,J} + u_1 \Psi_{2,J} = \left( QCX \right)_{1,J},\]

where \(\Psi_{0,J}\) is the value at \(x=0\) (or it can be defined inside the wall as a ghost cell, the same conclusion can be obtained).

Since Neumann boundary condition \(\frac{\delta \psi}{\delta x} = 0\) is imposed on the scalar potential (and this holds even after the Fourier transform is applied), we have \(\Psi_{0,J} = \Psi_{1,J}\) and thus we obtain

\[\left( l_1 + c_1 \right) \Psi_{1,J} + u_1 \Psi_{2,J} = \left( QCX \right)_{1,J},\]

i.e., the center diagonal value should be modified. The same story holds for the other boundary (\(c_{\text{itot}} \leftarrow c_{\text{itot}} + u_{\text{itot}}\)), and this treatment is implemented as:

126      TDM_C(   1) += TDM_L(   1);
127      TDM_C(itot) += TDM_U(itot);

Since the matrix solver is ready now, after the alignment is modified (from qcy, \(y\)-aligned to qcx, \(x\)-aligned):

92  parallel_transpose(jtot/2+1, itot, sizeof(fftw_complex), MPI_C_DOUBLE_COMPLEX, qcy, qcx);

the tri-diagonal matrix is solved:

129      for(int i=1; i<=itot; i++){
130        TDM_Q(i) = QCX(i, j);
131      }
132      my_zgtsv_b(itot, tdm_l, tdm_c, tdm_u, tdm_q);
133      for(int i=1; i<=itot; i++){
134        QCX(i, j) = TDM_Q(i);
135      }

which is repeated for all \(J\).

Now we have the answer of the Poisson equation, which is, however, defined in the wavespace (\(\Psi\)).

After the alignment of the variable is changed (from qcx, \(x\)-aligned to qcy, \(y\)-aligned):

147  parallel_transpose(itot, jtot/2+1, sizeof(fftw_complex), MPI_C_DOUBLE_COMPLEX, qcx, qcy);

we use the discrete inverse Fourier transform in order to retrieve the solution in the physical space (\(\psi\)):

149  fftw_execute(bwrd);

Finally, the alignment is again changed (to recover the same shape as the original scalar potential psi:

151  parallel_transpose(jtot, itot, sizeof(double), MPI_DOUBLE, qry, qrx);

and the result is stored to psi:

153  for(int j=1; j<=jsize; j++){
154    for(int i=1; i<=itot; i++){
155      PSI(i, j) = QRX(i, j)/(1.*jtot);
156    }
157  }

Note that a division by jtot is necessary because FFTW3 does not normalise the result.
