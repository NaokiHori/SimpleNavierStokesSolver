Description¶

Here the flow of the program is described briefly as well as the corresponding part of the code.

  1. Configuration

    Almost all parameters are set in exec.sh before launched. For simplicity, they are defined as local environmental variables (i.e., export XXX=value) and loaded later. For more detail of each parameter loaded here, see here.

  2. Initialisation

    Main function in src/main.c is called among others.

    MPI is launched, and time when the execution is started is recorded:

    16  MPI_Init(NULL, NULL);
17  double wtimes[2] = {0.};
18  wtimes[0] = parallel_get_wtime(MPI_MIN);

    Structures (e.g. fluid_t *fluid) and their members (e.g. fluid->ux) are allocated and initialised (initial values are assigned):

    20  param_t       *param       = param_init();
21  parallel_t    *parallel    = parallel_init();
22  fluid_t       *fluid       = fluid_init(param, parallel);
23  temperature_t *temperature = temperature_init(param, parallel);
24  statistics_t  *statistics  = statistics_init(param, parallel);

    Optimal diffusive term treatment (to minimise the overall computational load) is estimated:

    26  param_estimate_cost(param, parallel);

  3. Main loop

    First of all, time step size dt (based on the diffusive term treatment determined above) is decided:

    30    param_decide_dt(param, parallel, fluid);

    All equations (mass, momentum, and internal energy balances) are integrated in time for one step:

    32    integrate(param, parallel, fluid, temperature);

    Note

    See below for more details.

    Since the flow and temperature fields have been updated, time step (step) and time (time) should be incremented:

    34    param->step += 1;
35    param->time += param->dt;

  4. Logging and output

    Values to be monitored during the simulation (e.g., divergence, which should be small enough) are computed and output regularly (each param->log.rate simulation time units):

    37    if(param->log.next < param->time){
38      logging(param, parallel, fluid, temperature);
39      param->log.next += param->log.rate;
40    }

    Note

    This function is called when the current time param->time is larger than the scheduled time param->log.next. Once the logs are dumped, the schedule is updated to the next time, which is param->log.rate time units later.

    Similar schedule managements are done for save and statistics, which are called below.

    Flow fields are necessary for post-processings, visualisations, and restarting simulations; they are also dumped here regularly (each param->save.rate simulation time units):

    42    if(param->save.next < param->time){
43      save(param, parallel, fluid, temperature);
44      param->save.next += param->save.rate;
45    }

    Temporally-averaged statistics are collected during the simulation (each param->stat.rate simulation time units).

    47    if(param->stat.next < param->time){
48      statistics_collect(param, parallel, fluid, temperature, statistics);
49      param->stat.next += param->stat.rate;
50    }

    The program is terminated when the simulation reaches the maximum (free-fall) time units:

    52    if(param->time > param->timemax){
53      break;
54    }

    We also terminate the program when it has spent the maximum wall time:

    56    wtimes[1] = parallel_get_wtime(MPI_MAX);
57    if(wtimes[1]-wtimes[0] > param->wtimemax){
58      break;
59    }

  5. Clean-up and finalisation

    Final flow fields (velocity, pressure, and temperature) and collected statistics are saved:

    66  save(param, parallel, fluid, temperature);
67  statistics_output(param, parallel, statistics);

    Structures and their members are deallocated:

    69  statistics_finalise(statistics);
70  temperature_finalise(temperature);
71  fluid_finalise(fluid);
72  parallel_finalise(parallel);
73  param_finalise(param);

    Finally MPI is finalised:

    75  MPI_Finalize();

Description¶

This function integrates the governing equation for one time step.

  1. Update boundary values

    Boundary values (values on the walls in \(x\) direction, values of halo cells in \(y\) direction) of the velocities fluid->ux, fluid->uy and pressure :c-lang;`fluid->p` are assigned, which will be used to evaluate derivatives in the next steps:

    82    fluid_update_boundaries_ux(param, parallel, fluid->ux);
83    fluid_update_boundaries_uy(param, parallel, fluid->uy);
84    fluid_update_boundaries_p(param, parallel, fluid->p);

  2. Update temperature-related variables

    Based on the flow field (velocities), temperature field as well as buoyancy term are updated:

    86    if(param->with_temperature){
87      temperature_update_boundaries_temp(param, parallel, temperature->temp);
88      temperature_compute_force(param, parallel, temperature);
89      temperature_update_temp(param, parallel, rkstep, fluid, temperature);
90    }

  3. Predict velocity

    Using the buoyancy force, \(x\) and \(y\) velocities are updated

    92    fluid_update_velocity(param, parallel, rkstep, fluid, temperature);

    by integrating the momentum equation

    \[u_i^* = u_i^k + \left\{ \alpha^k \left( A_i^k + D_i^k \right) + \beta^k \left( A_i^{k-1} + D_i^{k-1} \right) + \gamma^k P_i^k \right\} \Delta t\]

    if the diffusive terms are treated explicitly, whereas

    \[\begin{split}\begin{aligned} & \delta u_i = \left\{ \alpha^k A_i^k + \beta^k A_i^{k-1} + \gamma^k \left( P_i^k + D_i^k \right) \right\} \Delta t, \\ & u_i^* = u_i^k + \left( 1 - \frac{\gamma^k \Delta t}{2 Re} \frac{\delta^2}{\delta x^2} \right)^{-1} \left( 1 - \frac{\gamma^k \Delta t}{2 Re} \frac{\delta^2}{\delta y^2} \right)^{-1} \delta u_i, \end{aligned}\end{split}\]

    if treated implicitly. See Numerical method for more details (e.g., the meaning of the variables).

  4. Compute scalar potential and correct velocity

    A scalar potential \(\psi\) is computed by solving a Poisson equation

    \[\frac{\delta ^2 \psi}{\delta x_i \delta x_i} = \frac{1}{\gamma^k \Delta t} \frac{\delta u_i^*}{\delta x_i}\]

    whose implementation is

    94    fluid_compute_potential(param, parallel, rkstep, fluid);

    to correct the velocity field \(u_i^*\), so that the incompressibility constraint is satisfied:

    \[u_i^{k+1} = u_i^* - \gamma^k \Delta t \frac{\delta \psi}{\delta x_i}\]

    whose implementation is

    96    fluid_correct_velocity(param, parallel, rkstep, fluid);

  5. Update pressure

    Finally the pressure is updated following

    \[p^{k+1} = p^k + \psi\]

    if the diffusive terms are treated explicitly, while

    \[\frac{\delta p^{n+1}}{\delta x_i} = \frac{\delta p^n}{\delta x_i} + \frac{\delta \psi}{\delta x_i} - \frac{\gamma^k \Delta t}{2 Re} \frac{\delta^2}{\delta x_j \delta x_j} \left( \frac{\delta \psi}{\delta x_i} \right)\]

    for implicit treatments:

    98    fluid_update_pressure(param, parallel, rkstep, fluid);

Note

These procedures are repeated for three times (since we use three -step Runge-Kutta scheme) to update the fields (values at \(n\) step is updated to ones at \(n+1\) step).
