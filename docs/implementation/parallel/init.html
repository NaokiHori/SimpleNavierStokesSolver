First the number of process is checked and assigned to mpisize:

10  MPI_Comm_size(MPI_COMM_WORLD, &mpisize);

Also the process ID of itself is checked and assigned to mpirank:

13  MPI_Comm_rank(MPI_COMM_WORLD, &mpirank);

Next, the neighbouring process IDs are assigned; the lower and upper process IDs are mpirank-1 and mpirank+1, respectively.

16  parallel->ymrank = mpirank-1;
17  parallel->yprank = mpirank+1;
18  if(mpirank == 0){
19    parallel->ymrank = mpisize-1;
20  }
21  if(mpirank == mpisize-1){
22    parallel->yprank = 0;
23  }

The schematic below shows an example when mpisize = 3:

Note that the values should be modified for the bottom and top processes, as can be seen in the sketch.

Finally, random seed is set for reproducibility here. By default, random numbers are only used to perturb the initial temperature field (in src/temperature/init.c):

38    // linear temperature distribution with perturbation
39    const double factor = 0.1;
40    const double *xc = param->xc;
41    double *temp = temperature->temp;
42    for(int j=1; j<=jsize; j++){
43      for(int i=1; i<=itot; i++){
44        double x = XC(i);
45        double r = -0.5+1.*rand()/RAND_MAX;
46        TEMP(i, j) = (TEMP_XP-TEMP_XM)*x+TEMP_XM+factor*r;
47      }
48    }

If we used the same seed for all processes, same perturbation is imposed and an artificial flow field can be generated; this is the reason why mpirank is used as the seed to create different perturbation in each domain.
