parallel_get_size¶

int parallel_get_size(
    const int num_total,
    const int mpisize,
    const int mpirank
);

Name

intent

description

num_total

mpisize

in

mpirank

in

rank of the process which calls this function

num_local

out(return)

number of local points

Description¶

As explained in the spatial discretisation, the domain (whose size is jtot) is split in \(y\) direction; this function determines the local (each processor’s) size of the domain (jsize) using the relation:

\[\text{jsize} = \left( \text{jtot} + \text{mpirank} \right) / \text{mpisize},\]

whose implementation can be found here:

8  int num_local = (num_total+mpirank)/mpisize;
9  return num_local;

Note that the division of an integer number truncates towards 0. For instance, when the number of grid points in \(y\) direction is 11 (jtot = 11) and the number of processors mpisize is 3, processor 0, 1, 2 (mpirank are 0, 1, and 2) have jsize = 3, 4, and 4, respectively:

Of course the summation of all jsize among all processes is equal to jtot to cover the whole domain.

out(return)

offset of the process

Description¶

The function parallel_get_size returns the number of grid points in each process. On the other hand, parallel_get_offset computes the offset by summing up the number of grid points in front of the process:

14  int offset = 0;
15  for(int i=0; i<mpirank; i++){
16    offset += parallel_get_size(num_total, mpisize, i);
17  }
18  return offset;

parallel_get_wtime¶

Definition¶

double parallel_get_wtime(
    const MPI_Op op
);

Name

intent

description

op

in

MPI operator

wtime

out(return)

current wall time

Description¶

The function parallel_get_wtime is a wrapper function of MPI_Wtime and returns the current wall time. In order to synchronize the result among all processes (indeed MPI_Wtime can return different results), MPI_Allreduce is used with an operation op (e.g., MPI_MAX or MPI_MIN in most cases).
