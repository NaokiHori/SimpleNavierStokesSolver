fileio.c¶

fileio.c contains functions which handle file input/output operations. Most functions are designed to write and read files in npy format.

The reasons why we adopt npy format in this project are as follows:

  1. Being binary

    We should avoid to save arrays in text format since the original data in floating points is modulated and also the file size tends to be further increased.

  2. Simplicity

    In order to recover the original data later, we need to store the datatype and shape as metadata with the raw data. In a npy file, these information are stored in the header part as descr and shape, respectively. Although HDF5 format offers the same and more general functionalities, they are too general and too complicated for the current simple purpose. Also we would like to reduce the number of external libraries to be simple.

  3. Generality

    NumPy is one of the most popular libraries for scientific computings (especially for post-processing and visualisation with Matplotlib), and npy format is its native binary format. Thanks to its simple structure, this format is easily accessible from other languages (e.g., Rust, MATLAB).

See also

Reading and writing npy headers rely on our library SimpleNpyIO, which is copied and used here.

Description¶

This function is a wrapper of a function fopen in C standard library stdio. fopen returns NULL when the file-open operation fails. This function checks it and output an error message.

41FILE *fileio_fopen(const char * restrict path, const char * restrict mode){
42  FILE *stream = fopen(path, mode);
43  if(stream == NULL){
44    char *error_message = generate_error_message(__FILE__, __LINE__, path);
45    perror(error_message);
46    common_free(error_message);
47  }
48  return stream;
49}

Note

A serial operation is assumed, i.e., only one process should call this function, and the same process should access to (read or write) the file, and the same process should call fileio_fclose.

This function is a wrapper of a function fclose in C standard library stdio. fclose returns EOF when the file-close operation fails. This function checks it and output an error message.

52int fileio_fclose(FILE *stream){
53  int retval = fclose(stream);
54  if(retval != 0){
55    char *error_message = generate_error_message(__FILE__, __LINE__, "");
56    perror(error_message);
57    common_free(error_message);
58  }
59  stream = NULL;
60  return 0;
61}

Note

A serial operation is assumed, i.e., only one process should call fileio_fopen, and the same process should access to (read or write) the file, and the same process should call this function.
