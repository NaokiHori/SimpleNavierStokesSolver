Description¶

Since the domain is wall-bounded in \(x\) direction, boundary condition should be imposed on the x boundaries. By default, Dirichlet boundary conditions are used:

16  for(int j=1; j<=jsize; j++){
17    TEMP(     0, j) = TEMP_XM;
18    TEMP(itot+1, j) = TEMP_XP;
19  }

The boundary values TEMP_XM and TEMP_XP are fixed in include/temperature.h to satisfy \(\Delta T = 1\).

Also halo cells in \(y\) direction are communicated here:

13  parallel_update_halo_ym(parallel, itot, MPI_DOUBLE, &TEMP(1, jsize), &TEMP(1,       0));
14  parallel_update_halo_yp(parallel, itot, MPI_DOUBLE, &TEMP(1,     1), &TEMP(1, jsize+1));

where parallel_update_halo_ym is responsible for the communication with the lower process:

while parallel_update_halo_yp is responsible for the communication with the upper process:

Note that values only between i=1 and i=itot (itot components) are communicated in each function, i.e., boundary values are not communicated, which is because their values are fixed and thus not necessary to be exchanged.

Also see src/parallel/halo.c.
