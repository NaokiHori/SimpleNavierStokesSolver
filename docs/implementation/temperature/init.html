Description¶

A temperature structure and its members are allocated:

58  allocate(param, parallel, &temperature);

and values are initialised when the simulation is started from the beginning, or values are loaded from a directory (char param->dirname_restart, see param) to restart the run:

60  init_or_load(param, parallel, temperature);

Description¶

Velocity and pressure fields are initialised.

When we want to start the simulation from the beginning (i.e., param->load_flow_field is false), initial conditions are assigned here:

38    // linear temperature distribution with perturbation
39    const double factor = 0.1;
40    const double *xc = param->xc;
41    double *temp = temperature->temp;
42    for(int j=1; j<=jsize; j++){
43      for(int i=1; i<=itot; i++){
44        double x = XC(i);
45        double r = -0.5+1.*rand()/RAND_MAX;
46        TEMP(i, j) = (TEMP_XP-TEMP_XM)*x+TEMP_XM+factor*r;
47      }
48    }

Otherwise (i.e., param->load_flow_field is true), values are loaded from a restart file:

35    fileio_r_p_like_parallel(param->dirname_restart, "temp", param, parallel, temperature->temp);

where the temperature field is loaded from a npy file (see src/fileio.c for details).

Note that flow fields excluding halo cells and walls are set here, and these edge values are set subsequently:

51  temperature_update_boundaries_temp(param, parallel, temp);
